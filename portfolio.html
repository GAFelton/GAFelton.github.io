Project: Covid-19 Dashboard

Description

This COVID-19 tracking dashboard website pulls up-to-date covid-19 information via 3rd party APIs then present the data in a cohesive, simple dashboard. Data is segmented into a few categories: Statistical Data, Interactive Map, and Latest News. The landing page & dashboard menu make it easy for users to see data for different states in the USA, or even the whole country. The webpage is responsive & accessible to screen-reader users. The landing page features a clickable map.

Link to Deployed Project | Link to GitHub Repo

Tools Used:

  • HTML
  • CSS / Zurb Foundation
  • JavaScript / jQuery
  • SimpleMaps.com

APIs Used:

  • TrackCorona.live API (City Level Data)
  • Google Map API
  • COVID Tracking Project API (Historical US Data & Historical State Data)
  • New York Times API (Article Search)

User Story:

AS A person concerned about COVID19 in my area,
I WANT to easily access up-to-date information in one page
SO THAT I can make informed decisions when planning my day.

Project Details:

This was a 6-person group project. For all of us, it was our first time collaborating through GitHub. I took a leadership role in building this webpage: Once we decided what we were doing, I wrote up our project proposal & built the initial wireframe for the dashboard.
On the front-end team, I learned Zurb Foundation & built the dashboard page from the ground-up. I found & hooked up the clickable map for the landing page (index.html). I made the site responsive and did 1/2 of the work to make it screen-reader accessible.
I also put a lot of effort into project management: I managed the Kanban board & GitHub issues, and made sure tasks were divyed up. I wrote up a git workflow & placed it in the GitHub wiki, and also spent time 1:1 with my teammates making sure everyone understood branching & making pull requests.

Project: Weather Dashboard

Description

Weather Dashboard - Created a webpage to display current weather conditions and 5 day forecast based on user input. Used the 3 aspects of the OpenWeatherMap API: Current Conditions, UV Index, and the 5 Day Forecast. Stored previous searches in local storage, and populated a list of previous searches.

Link to Deployed Project | Link to GitHub Repo

Tools Used:

  • HTML
  • CSS / Bootstrap
  • JavaScript / jQuery

APIs Used:

  • OpenWeatherMap API:
    • Current Conditions
    • UV Index
    • Five Day Forecast

User Story:

AS A traveler
I WANT to see the weather outlook for multiple cities
SO THAT I can plan a trip accordingly.

Project Details:

I have two main API calls - for the current weather data and the 5 day forecast. The UV index is also a separate API call nested within the current weather data call, because it requires coordinates generated from the first API. Nearly all page elements are dynamically generated - the page first loads with just a search bar.
Previous searches are stored in localStorage, and if the user revisits the page their most recent search will populate the page. Users have the option to delete this list of previous searches. Additionally, if the list starts getting too long, it will collapse the list and allow access via a button.
I also implemented error messaging for 404 or other problems, and stop users from making blank searches. If a user searches for the same city twice, it will not populate the previous searches list twice.
